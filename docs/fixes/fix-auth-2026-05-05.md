### LOT B — M-4 (audit RLS) — Rate-limit DB `producer_interests`

- Trigger `trg_producer_interests_rate_limit` BEFORE INSERT.

**Test fonctionnel** :

- 4e insert → rejeté avec `23P01` + hint `producer_interests_rate_limit`.

**Justification** : pas de mécanisme cross-session permettant d'invalider le cookie HMAC d'un user **autre** que celui qui déclenche le changement (le cookie est isolé au browser cible). ### Drift `version_id` auto-généré par `mcp__supabase__apply_migration`

**Constat** : les migrations apply via MCP `apply_migration` sont tracées dans `supabase_migrations.schema_migrations` sous un `version_id` **auto-généré** au moment de l'apply (timestamp de l'apply, pas le préfixe du fichier local). Exemples observés ce 2026-05-05 :

|---|---|
| `20260505100000_audit_rls_lot_1_2_harden_security_definer_acls.sql` | `20260505112936` |
| `20260505100100_audit_rls_lot_3_4_optimize_rls_perf.sql` | `20260505113121` |
| `20260505100200_audit_rls_lot_5_fix_storage_policies_select.sql` | `20260505113241` |
| `20260505100300_audit_rls_lot_7_harden_rls_medium.sql` | `20260505113343` |
| `20260505100400_audit_rls_lot_8_cleanup_rls_low.sql` | `20260505113449` |
| (LOT 0 ce jour) `audit_rls_regression_new1_search_path_t241` (pas de fichier local) | `20260505115433` |

**Posture assumée** : drift accepté comme comportement attendu MCP. Aucun impact runtime (Postgres exécute les DDL via le content du `statements`, pas via le `version_id`). Re-réalignement à la `LOT 0bis` non nécessaire pour ces entrées (elles ne correspondent pas à des fichiers locaux numérotés sous le format YYYYMMDDHHMMSS du flow `supabase migration up --linked`).

**Action de suivi** (forward-looking, hors scope) : si un futur passage à `supabase migration up --linked` est adopté, créer les fichiers locaux correspondants pour ces migrations apply via MCP afin que la table de tracking soit cohérente avec le repo.
