# Page /cgu (version complète) — 2026-05-06

Cinquième vague des pages P0 légales (audit architecture 2026-05-06).
Remplacement du placeholder minimaliste de `/cgu` (créé lors du commit
`/mentions-legales` du 2026-05-06) par une **page CGU complète et
fonctionnelle** en 14 articles.

CGU vs CGV : les CGU régissent l'**usage du Site** (compte,
comportement, contenus utilisateurs, modération, propriété
intellectuelle, responsabilités), les CGV régissent la **transaction
commerciale** Consumer ↔ Producer. Cette session traite uniquement
les CGU. La page `/cgv` reste en placeholder minimaliste, à traiter
en session dédiée.

Page **complète et fonctionnelle dès maintenant**, avec bandeau
placeholder violet TRÈS visible signalant la nécessité d'une
validation par un juriste avant le lancement officiel. Pattern aligné
sur `/mentions-legales` (livré la veille).

Robots `index, follow` (page juridique standard publique). Server
Component pur, zéro JS hors hydration Next.

## Trade-offs et décisions autonomes

- **Pattern visuel** : copié à 100% de `/mentions-legales` plutôt que
  réinventé. Cohérence inter-pages juridiques + maintenance facilitée.
- **Bandeau placeholder global** plutôt que ciblé : le contenu CGU
  est rédigé sans placeholders inline (à l'inverse de
  `/mentions-legales` qui a 11 placeholders dans `EDITOR_INFOS`). Le
  bandeau global signale la nécessité de validation juriste sans
  truffer le texte de marqueurs violets qui le rendraient illisible.
- **Article 6.4 — Avis** rédigé au présent et non au futur, suite
  audit LOT 0 (feature implémentée).
- **Article 9.1 — Rôle d'intermédiaire** : formulation prudente
  « TerrOir n'est pas partie aux contrats de vente », essentielle
  pour le statut de plateforme de mise en relation et la limitation
  de responsabilité Stripe Connect.
- **Article 11 — Signalement** : ancrage LCEN art. 6 + DSA. Pas de
  formulaire dédié, redirection vers `/contact` (cohérent avec le
  pattern adopté pour les signalements RGPD).
- **Pas de checkbox CGU à l'inscription** : signalé en doc fix mais
  hors scope de cette session (toucherait au flow auth consumer +
  producer + DB schema). Trade-off : on ouvre les CGU en index
  Google avant que la chaîne d'acceptation soit branchée — risque
  juridique modéré (contenu CGU = informatif, pas opposable tant
  que pas accepté).
- **`robots: index, follow`** : la page placeholder précédente était
  en noindex (cohérent quand le contenu n'existait pas). Maintenant
  que la page est complète et utile, indexation activée pour SEO
  et trust signals (footer juridique standard).
- **Pas de toggle « ouvrir/fermer chaque article »** : longueur de
  lecture importante (1500+ mots) mais layout colonne unique avec
  espacement généreux + ancres + retour haut de page suffisent.
  Cohérent avec `/mentions-legales`. Pour des CGV plus denses on
  pourra introduire un sommaire collant.
- **Pas de réécriture de la feature avis** : strict respect du
  scope. La doc fix mentionne juste les surfaces identifiées.
