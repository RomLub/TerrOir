## Doctrine pour les audits futurs

- `npm audit fix` non-breaking dans commit dédié (jamais bundle avec d'autres changements).
- `npm audit fix --force` JAMAIS sans arbitrage explicite Romain — peut casser le build OU introduire breaking changes API.
- Audit npm à passer en CI (à câbler post-Live, T-296 ou item dédié) — `npm audit --audit-level=high` → exit 1 si nouvelles vulns high+.
